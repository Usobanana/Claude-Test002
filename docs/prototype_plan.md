# プロトタイプ開発プラン

## 方針

- **Prototype1**：ソロで基本ゲームループが遊べる状態
- **Prototype2**：2人で同じ拠点・フィールドに入れるマルチプレイ対応
- 他のゲームクリエイターに見せることを想定し、アセットクオリティも意識する

---

## 技術スタック

| 用途 | 採用技術 |
|---|---|
| エンジン | Unity（URP） |
| 入力 | Unity Input System |
| ナビゲーション（敵AI） | Unity AI Navigation（NavMesh） |
| マルチプレイ（Prototype2） | Photon Fusion 2（小規模開発は無料枠で対応） |
| アセット | Synty Studios（有料）・Asset Store無料アセット・AI生成素材 |

## Prototype1：ソロプレイ

### ゴール
**拠点 → フィールド → ボス討伐 → リザルト → 拠点** の1ループが遊べること

### フェーズ構成

#### Phase 1：プロジェクト基盤 ✅
- シーン構成の設計・作成（TitleScene / BaseScene / FieldScene / ResultScene）
- シーン遷移管理システム
- キャラクターデータ構造の設計（ステータス、Lv/EXP）
- InputSystem のアクションマップ設定（移動・スキル・回避）

#### Phase 2：キャラクター・移動 ✅
- 戦士キャラクターの実装（トップダウン移動）
- WASD / コントローラー入力対応
- スタミナシステム（回避・ダッシュ、時間回復）
- キャラクターアニメーション（Synty or 仮素材）

#### Phase 3：バトルシステム ✅
- オートアタック（円形範囲、向きによるターゲット選定）
- アクティブスキル3枠（クールタイム）
- ウルトスキル1枠（クールタイム）
- ダメージ計算式の実装（攻撃力 × 倍率 × 防御補正 × 会心補正）

#### Phase 4：エネミーシステム ✅
- ザコエネミーAI（NavMesh移動・攻撃）
- ボスAI（基本パターン・HPフェーズ）
- エネミーのHP・ダメージ処理

#### Phase 5：拠点・ゲームループ ✅
- 拠点シーン（ギルドでクエスト受理・出口からフィールドへ）
- フィールドシーン（ザコ → ボスエリアの構成）
- リザルト画面（クリア / 失敗、報酬表示）
- ゲームループ完成（拠点 → フィールド → リザルト → 拠点）

#### Phase 6：UI・HUD ✅
- HUD（HP・スタミナ・スキルボタン・クールタイム表示）
- 拠点メニューUI（ギルド画面、クエスト一覧）
- リザルト画面UI

#### Phase 7：アセット統合・ポリッシュ ✅（進行中）
- [x] Synty POLYGON Fantasy Kingdom インポート・URP変換
- [x] Synty Animation Sword Combat インポート（プレイヤー用）
- [x] Synty Animation Goblin Locomotion インポート（敵用）
- [x] プレイヤーにPOLYGONRig_01モデル適用
- [x] PlayerAnimatorController作成（Idle/Run/Attack/Dodge/Death）
- [ ] 敵キャラクターモデル・アニメーション適用
- [ ] 環境アセット配置（BaseScene・FieldScene）
- [ ] SE・BGMの仮実装
- [ ] カメラ調整・エフェクト

## Prototype2：マルチプレイ対応

### ゴール
**2人が同じ拠点に集まり、一緒にフィールドをクリアできること**

### 前提
- Prototype1 のソロ機能をベースに追加実装
- ネットワーク：**Photon Fusion 2**

### フェーズ構成

#### Phase 8：ネットワーク基盤
- Photon Fusion 2 の導入・設定（Photon Dashboardでアプリ登録）
- ホスト（リーダー）作成 / ゲスト参加フローの実装
- メニューからリーダーの拠点へ移動する機能

#### Phase 9：拠点のマルチプレイ対応
- 拠点シーンの同期（複数プレイヤーが同じ空間に存在）
- クエスト受注の全員HUD共有
- 出発確認システム（全員が出口で「出発」を押すとフィールドへ）

#### Phase 10：フィールドのマルチプレイ対応
- プレイヤー移動・攻撃の同期
- エネミーHPの同期
- ボス討伐判定の共有

#### Phase 11：リザルト・帰還の同期
- リザルト画面の同期表示
- 全員がリーダーの拠点に戻る処理
